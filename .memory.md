## üìã APPLICATION OVERVIEW

**Mob Hunter v3.0** is a complete MMORPG auto-combat bot for **Silkroad Online**. It uses computer vision, pattern matching, and color detection to automatically detect mobs, filter out pets, and engage in combat.

### Core Purpose
- **Real-time mob detection** using screen capture and image processing
- **Center-out targeting** - prioritizes mobs closest to screen center
- **Binary health detection** - ALIVE or DEAD based on red pixel count
- **Pet filtering via classification** - No classification = PET (100%)
- **Automated combat** with skill rotation
- **Automatic buffer system** - Runs every 2 minutes
- **Transparent click-through overlay** - See-through visualization
- **Comprehensive logging** with session-based logs

### Key Features
1. **Center-Out Targeting** - Attacks closest mobs to screen center first
2. **Binary Health Detection** - Simple ALIVE/DEAD (>50 red pixels = alive)
3. **Pet Classification Filtering** - Has classification icon = MOB, No icon = PET
4. **Position Caching** - Remembers recently checked positions to avoid duplicates
5. **Automatic Buffing** - F2‚Üí1‚Üí2‚Üí3‚Üí4‚ÜíF1 every 2 minutes
6. **Transparent Overlay** - Click-through, fully transparent background
7. **Global Hotkeys** - CapsLock (pause/resume), Tab (toggle overlay)
8. **Session Logging** - Creates timestamped log directories

### Step-by-Step Flow

1. **Screen Capture**
   - Captures full screen (1920x1080) using MSS

2. **Floating Name Detection**
   - Converts to grayscale

   - **Calculates distance from screen center**
   - **Sorts by distance (closest first)**
   - Deduplicates nearby detections

3. **Pre-Filtering**
   - Checks position cache (skip if recently checked)
   - Uses distance-sorted list

4. **Click and Verify** (Max 3 per cycle)
   - Clicks position (below text, +25px)
   - Waits 0.25s
   - Captures nameplate region

   - Execute skill rotation: 1, 2, 3, 4
   - 0.6s delay between skills
   - Every 1s: Check if mob still alive
   - Early stop if DEAD detected

7. **Buffer Check**
   - Every cycle: Check if 120s passed

## üéØ KEY FEATURES

### 1. Center-Out Targeting
**Priority:** Distance from screen center (closest first)

**Rationale:**
- Mobs closest to center are usually closest to player
- Faster engagement
- More natural targeting

**Implementation:**
```python
# Calculate distance from screen center
distance = math.sqrt((cx - center_x)**2 + (cy - center_y)**2)

# Sort by distance (ascending)
detections.sort(key=lambda d: d['distance_from_center'])
```

### 2. Binary Health Detection
**Logic:** >50 red pixels = ALIVE, ‚â§50 = DEAD

**Rationale:**
- Simple, fast detection
- No percentage calculation needed
- Works reliably for combat decisions

**Implementation:**
```python
def is_mob_alive(self, nameplate):
    hsv = cv2.cvtColor(nameplate, cv2.COLOR_BGR2HSV)
    # Detect red pixels (health bar)
    red_pixels = cv2.countNonZero(red_mask1) + cv2.countNonZero(red_mask2)
    return red_pixels > Config.RED_PIXEL_THRESHOLD  # 50
```

### 3. Pet Filtering via Classification
**Logic:** Has classification icon = MOB, No icon = PET (100%)

**Rationale:**
- Mobs ALWAYS have classification icon (General/Champion/Giant/Unique)
- Pets NEVER have classification icon
- 100% accurate filtering

**Implementation:**
```python
# Check for any classification color
has_yellow = yellow_pixels > 50    # Giant
has_purple = purple_pixels > 50    # Champion
has_red = red_pixels > 100         # Unique

# If no classification colors detected = PET
if not has_yellow and not has_purple and not has_red:
    return None  # No classification = PET
```

**Schedule:**
- At startup (before first cycle)
- Every 120 seconds (2 minutes)
- When resuming from pause

**Sequence:**

4. Press 3 ‚Üí Wait 1 sec
5. Press 4 ‚Üí Wait 1 sec
6. Press F1

**Duration:** ~3.4 seconds every 2 minutes (97.2% uptime)

### 5. Transparent Click-Through Overlay
**Features:**
